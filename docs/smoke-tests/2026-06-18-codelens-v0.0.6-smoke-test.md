## Bugs Found (3 — all confirmed)

### BUG-1: `npx --no-install tsc` resolves to wrong package

`npx tsc` looks up the npm package literally named `tsc` (a Haskell Time Series Compiler, v2.0.4). The correct binary lives in the `typescript` package.

**Affects:**
- `agents/codelens-reviewer.md` Phase 2 `p2-tsc` command
- `skills/doctor/SKILL.md` check #11

**Fix:** `npx --no-install --package=typescript tsc ...` (explicit package) or `node_modules/.bin/tsc ...` (direct).

### BUG-2: Biome JSON field name

Biome v2.2.x JSON output uses `"location":{"path":"<file>"}` (string), not `"location":{"path":{"file":"<file>"}}` (nested object). The `r3-complexity` grep pattern `"file":"[^"]+"` will never match.

**Affects:** `agents/codelens-reviewer.md` `r3-complexity` command + parsing rule in P2 post-processor.

**Fix:** Change `rg -o '"file":"[^"]+"'` → `rg -o '"path":"[^"]+"'`. Update parser regex accordingly.

### BUG-3: Biome `--quiet` flag does not exist

Biome v2.2.x has no `--quiet` flag. `biome lint <path> --reporter=summary --quiet` errors with `no such flag: --quiet`, producing no useful output.

**Affects:** `agents/codelens-reviewer.md` `p2-biome` command (pre-existing since v0.0.4).

**Fix:** Drop `--quiet` from the command.

## Observations

### Positive

1. **All 5 context-mode MCP tools respond correctly** — ctx_stats, ctx_execute, ctx_execute_file, ctx_search, ctx_batch_execute, plus ctx_index. Phase 0 + Phase 1+2 plumbing is healthy.

2. **rg-based signal pipelines work end-to-end** — r1-loc, r2-finding-density, r4-centrality, all p2-* sec/quality/a11y patterns produced expected output on the fixture.

3. **Phase 3 rg fallback chains all work** when ast-grep is missing. The `(... || ...)` shell construct and `|| echo none` terminator correctly handle the no-sg case.

4. **Domain-aware filtering** works — only the requested ast-grep patterns would be pushed into `cmds`.

5. **Doctor correctly emits install hint** for ast-grep. The warn-only behavior is right.

### Negative

1. **Three commands are broken** in Phase 2 (p2-tsc, p2-biome, r3-complexity). All stem from guessing CLI/JSON shape from memory rather than verifying against installed tool versions. This matches the prediction in the v0.0.6 implementation review ("5 of 7 bugs were in shell string construction or field-name guessing").

2. **All three bugs are silent.** They produce empty output or errors that the `|| echo '*-not-available'` terminator swallows. The agent sees "biome unavailable" when biome is actually installed and working. This degrades the risk-score weighting (complexity=0 for all files) and the quality mapping (no biome lint/a11y signals).

3. **No timeout on tsc command.** Even after fixing the package name, `tsc --noEmit` on a large repo can hang. Should add `timeout 30`.

## Recommendations

### Immediate (v0.0.6.1 patch)

1. Fix BUG-1 (tsc package name) — affects both doctor + Phase 2.
2. Fix BUG-2 (biome JSON `path` vs `file`) — affects risk-score complexity signal.
3. Fix BUG-3 (drop biome `--quiet`) — affects biome summary capture.
4. Add `timeout 30` to tsc command in Phase 2.

### Short-term (v0.0.7)

5. Add a fixture-based smoke test to CI. Run `/codelens:review` on `examples/smoke-fixture.*` after every change to the agent. Asserts: ≥N findings per domain, no empty biome/tsc output.
6. Doctor check #11 should call `node_modules/.bin/tsc --version` (project-local) AND `npx --package=typescript tsc --version` (fallback) with proper package binding.
7. Add explicit "tool unavailable" detection in Phase 2 — if biome returns empty AND biome is installed (per doctor), treat as parse failure not "not available".

### Long-term (v0.0.8+)

8. Add ast-grep to the recommended install list in README — currently in doctor warnings only.
9. Cache biome + tsc results per-session to avoid recomputing on every review.
10. Cross-validate biome findings against rg findings — if biome finds 0 a11y violations but rg finds `<button>` matches, flag the discrepancy.
