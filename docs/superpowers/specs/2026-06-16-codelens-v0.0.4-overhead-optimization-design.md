# Codelens v0.0.4 — Tool-Substitution Optimization Design

**Date:** 2026-06-17 (revised from 2026-06-16 prompt-side optimization design)
**Target version:** v0.0.4
**Status:** Design — brainstorm complete, awaiting implementation

## Context

codelens v0.0.3 ships a single `codelens-reviewer` agent that performs multi-domain
code review using hand-written `rg` patterns inlined in the agent prompt. The repo
already contains `reports/codelens-reviewer-tool-validation.md` (committed in
`ed720df`) evaluating 10 purpose-built tools with benchmark evidence — Biome, semgrep,
jscpd, tokei, repotoire, osv-scanner, PHPStan, DCM, knip, and composer-dependency-analyser.

**Original v0.0.4 plan** was prompt-side micro-optimization (delete Phase 0 preflight,
extract report template, consolidate rg via alternation). Implemented and smoke-tested,
it produced a -308 token (-5.7%) prompt reduction. Reverted after recognizing this
ignored the bigger lever the tool-validation report points at: **delegate pattern-matching
to purpose-built tools** instead of doing it with hand-written rg patterns.

**Problem being solved (revised):** the agent does work that better tools already do
faster and more accurately. Hand-written rg patterns for quality/a11y duplicate what
Biome does in one pass (56× faster than ESLint per the report). Dead-code analysis via
rg is impossible — knip does it via AST analysis. Continuing to maintain rg patterns
for these jobs is reinventing wheels poorly.

**Intended outcome:** v0.0.4 substitutes Biome (quality + a11y) and knip (dead code)
for the corresponding rg pattern blocks, with rg fallback when the tools aren't
installed. Result: fewer round-trips, better findings (Biome catches SVG a11y, knip
catches unused deps — neither possible with rg), and an honest JS/TS language scope.

**Final v0.0.4 tool stack: 3 tools** (Biome, fallow, semgrep) plus rg as the existing primary. All install via standard package managers (npm/pip) in under 2 minutes each, all verified working on this machine.

**Why fallow collapsed the stack:** fallow is codebase intelligence for TS/JS covering unused code, duplication, circular deps, complexity hotspots, and architecture boundaries in one Rust-native sub-second tool. Its own benchmarks show 16-20× speedup over jscpd for duplication alone. Adding fallow made knip and jscpd redundant. repotoire (the architecture tool we couldn't install) is also covered by fallow's boundary-violation detection.

**Per-domain coverage after tool substitution:**

| Domain | Primary tool | Fallback (tool missing or non-JS/TS) |
|---|---|---|
| Security | semgrep (SAST, 87% TPR per OWASP benchmark) | rg patterns |
| Architecture | fallow (circular deps + boundary violations + complexity hotspots) | rg patterns |
| Quality (style/lint) | Biome (491 rules, 56× faster than ESLint) | rg patterns |
| Quality (dead-code) | fallow `dead-code` (AST-based, unused files/exports/deps) | skipped (rg can't do it — document the gap) |
| Quality (duplication) | fallow `dupes` (16-20× faster than jscpd) | skipped (rg can't do it — document the gap) |
| Accessibility | Biome (15+ HTML/JSX a11y rules) | rg patterns |

## Design Decisions (from brainstorm)

### 1. Missing-tool handling: detect and fallback to rg

Agent probes for Biome and knip at Phase 2 start via `command -v` (shell builtin, no
subprocess spawn). Present → use the tool. Missing → fall back to the existing rg
patterns for that domain. Users with the tools get the better+faster path; users
without still get a working review.

### 2. Tool output handling: pipe via ctx_execute

Biome and knip both run via `ctx_execute` (shell). Outputs over ~5KB auto-index; only
previews return to the agent's context. Agent uses `ctx_search` to pull specific
findings. Consistent with the existing Phase 2 rg pattern. Verified sizing:

- dockerize-react-app (8 files): Biome JSON 459 bytes, knip JSON 1,049 bytes — inline OK.
- portfolio/components (36 files): Biome JSON 11,898 bytes — auto-indexed, previews only.

### 3. Language scope: explicit JS/TS only

v0.0.4 declares JS/TS-only support. For Python/Go/Rust/PHP, the agent emits a clear
"language not supported, partial rg-only review" note in the report. Honest scope, no
false promise. Multi-language expansion (PHPStan for PHP, ruff for Python, etc.) is
a v0.0.5+ concern.

### 4. Phase 2 smoke-test methodology: empirical 4-config comparison

Before writing the final Phase 2 prose, run 4 configurations against dockerize-react-app
(cheap shape: security + quality, scope `./src`):

| Config | Phase 2 uses | Hypothesis |
|---|---|---|
| 1 (baseline) | rg patterns only (current v0.0.3) | reference — already have T_prompt=5420 |
| 2 | Biome for quality+a11y, rg for security+architecture | fewer calls, more a11y findings |
| 3 | knip for dead-code (quality subset), rg for everything else | catches unused deps rg can't |
| 4 | Biome + knip + rg fallback for security+architecture | best coverage |

Record T_prompt + findings count + qualitative signal for each. Pick the winner.

## Hard Constraints (preserved from v0.0.3)

- Single-pass per hotspot file
- Domain-aware (only run requested domains)
- Scope-aware (every command targets `scopePath`)
- Severity-first ordering
- Evidence-backed findings (file:line + snippet)
- Cross-domain dedup
- Exclusions honored (`.claude/codelens-exclusions.json`)
- Append-only log (`.codelens/reviews.json`, 6-field schema)

## Phases

### Phase 2 (core change): Biome + knip integration with rg fallback

The Phase 2 prose rewrites the per-domain rg blocks:

- **quality + a11y domains** (JS/TS projects only): probe for Biome. Present →
  `biome lint <scopePath> --reporter=summary` via `ctx_execute` for counts,
  `--reporter=json` via `ctx_execute` for evidence. Map Biome rules to codelens
  severities. Missing → existing rg quality/a11y patterns.
- **quality domain** (dead-code subset, JS/TS only): probe for knip. Present →
  `knip --reporter json` via `ctx_execute`. Missing → skip dead-code analysis
  (rg can't do it; document the gap in the report).
- **security + architecture domains**: unchanged, still rg-based.

### Phase 0: Graceful degradation (carry over)

Delete the 3-check upfront preflight. Replace with a 2-line instruction: "if any
required tool returns an error during use, halt with the matching install hint."
Research-backed (Claude Code runtime knows what's loaded; preflight is ceremony).

### Phase 4: Extract report template (carry over)

Move the inlined 4.2KB report template to `references/report-template.md`. Agent reads
it via `ctx_execute_file` with `intent: "codelens:report-template"` at Phase 4 start.
Deterministic ~284 token prompt reduction.

### Report: Language-scope declaration

Add a `**Supported languages:** JavaScript/TypeScript` line to the report header. If
the reviewed code is primarily non-JS/TS, emit a `## Language Support Note` section
explaining the partial coverage.

## Verification

Per phase:
1. **T_prompt before/after** (deterministic — `wc -c` on the agent body).
2. **One smoke run** on dockerize-react-app cheap shape to confirm the agent produces
   a valid report. No multi-run median (established earlier that agent non-determinism
   makes N_tools/findings too noisy to benchmark meaningfully).
3. **Code inspection** confirms the structural change (e.g., "Biome call present,
   rg fallback present").

Phase 2 additionally: run the 4-config empirical comparison before finalizing.

## Risks

| Risk | Mitigation |
|---|---|
| Biome rule → codelens severity mapping is subjective | Document the mapping table in the agent; treat as initial best-guess, refine in v0.0.5 |
| knip requires package.json context — fails on non-JS targets | Probe result handles this; knip-missing falls back to "dead-code analysis skipped" |
| Agent prompt grows (both tool integration AND rg fallback paths) | Phase 4 template extraction offsets this; net prompt change to be measured |
| Biome's 491 rules produce noise the agent must triage | Triage via `ctx_search` on the auto-indexed output; agent reasons over previews, not full output |
| User has old Biome (pre-v2) with different rule names | Documented minimum version (Biome 2.x); README notes the requirement |

## Out of Scope

- Multi-language support (PHP, Python, Go, Rust) — v0.0.5+
- semgrep, osv-scanner, jscpd, tokei, repotoire integration — install cost too high for a plugin prerequisite
- Switching from single-agent to multi-agent architecture
- New domains beyond security/architecture/quality/a11y
