# Codelens v0.0.9 — Phase 4 Gate-Hardening Design

**Date:** 2026-06-19
**Status:** Design (awaiting implementation plan)
**Supersedes portion of:** `2026-06-18-codelens-v0.0.8-architecture-design.md` §4.2 (Deterministic validation gate) — that section designed the gates; this spec makes the agent actually run them.

---

## Context

### The problem

The 2026-06-19 smoke test (`docs/smoke-tests/2026-06-19-codelens-v0.0.8-smoke-test.md`, my-portfolio, 151 files) proved three CRITICAL issues:

1. The agent wrote a malformed `.codelens/reviews.log` entry with wrong keys (`scopeTarget`, `domains`, `outputFile`, `topFindings`, `notes`) — none of which are in the current 11-field schema (`ts`, `scope`, `crit`, `high`, `med`, `low`, `info`, `report`, `v`, `tokIn`, `tokOut`). The on-disk validator would reject it.
2. The agent never loaded `templates/report.md` — `grep 'templates/' transcript` returned 0 hits. It reinvented the report format (letter-grade scorecard instead of severity-count table).
3. The agent made **zero** `ctx_execute` calls during the entire run. The validation gates designed in the v0.0.8 spec (`scripts/validate-report.sh`, `scripts/validate-entry.js`) exist on disk but were never invoked.

### Root cause

The agent is not defying instructions or losing context. It skips the gates because the Phase 4 gate calls are written as **pseudo-syntax in prose**, not real tool invocations. Specifically in `agents/codelens-reviewer.md`:

- **Step 1 (line 302)** writes `ctx_execute_file path: "..." intent: "..."` — not valid JSON for the tool. It's paraphrase bait.
- **Step 4 (line 334)** writes `Call ctx_execute language="shell" code="bash scripts/validate-report.sh..."` — same pseudo-syntax.
- **Step 6 (line 358)** asks the agent to read `validate-entry.js`, inline it, and substitute the entry — a 3-step composition with no worked example.
- The `<constraints>` block (line 394) undercuts Step 1 with a weaker paraphrase: *"Always consult templates/README.md + templates/reviews-entry.json"* — "consult" is satisfiable from training data without ever issuing the call.

**The cross-version pattern:** real tool-call syntax in fenced blocks fires (Phase 0's `ctx_stats()`); pseudo-syntax in prose doesn't (Phase 3 `cat`, v0.0.8 gates). The v0.0.8 smoke test is the latest instance of this pattern.

### Intended outcome

Make the Phase 4 gates fire reliably, so that:
- Every review loads the three output-contract templates before writing any structured output.
- Every review runs both validators before appending to `.codelens/reviews.log`.
- Output drift becomes a loud, recoverable failure (validator `FAIL`), not a silent one (malformed entry on disk).

This is a **discipline release**, not a structural rewrite. The agent stays single-pass and stateless.

---

## Design

### Phased approach

- **Phase A (this release, v0.0.9):** Prose hardening — convert gate calls from pseudo-syntax to real tool-invocation JSON, add required `STATUS:` markers, hoist a preflight banner above Step 1.
- **Phase B (next release, v0.0.10):** Hook enforcement — move gate invocation into a Claude Code `PreToolUse:Write` hook so the harness blocks any append to `reviews.log` that wasn't preceded by the three markers in the transcript.

Phase A is shipped first and smoke-tested alone. Phase B is designed but not implemented in this spec; it's the fallback if Phase A prose is still ignored, and the long-term guarantee.

---

### Phase A — Prose hardening

**Goal:** Make the gate calls real tool invocations with observable footprints the smoke test can grep for.

**Strategy:** Three levers applied to every gate:
1. **Real JSON tool-call blocks** (copy-paste, not paraphrase).
2. **Required `STATUS:` markers** printed to the transcript after each gate succeeds.
3. **A preflight banner** at the top of Phase 4 listing the gates, exact calls, and required markers — framing them as an observable contract with the smoke-test harness, not internal hygiene.

Plus: remove the competing `<constraints>` paraphrase that undercuts Step 1, and fix a latent frontmatter bug (`ctx_stats` missing from `tools:`).

### Phase B — Hook enforcement (designed, not implemented in v0.0.9)

**Goal:** Move gate invocation into the harness so the prompt cannot bypass it.

**Enforcement point:** `PreToolUse:Write` scoped to the `reviews.log` path. The hook greps the transcript for the three `STATUS:` markers; if any is missing, it returns `{decision: "block", reason: ...}` and the Write is refused. The append is the irretrievable side-effect — that's the right place to enforce. The report Write is allowed earlier (Step 3) because Step 4's validator may FAIL and the agent re-Writes; blocking the report Write would break that retry loop.

A second `Stop` hook writes `/tmp/codelens-gate-verdict.json` for the smoke test to read in headless mode, capturing silent drift.

**Hook code and `settings.local.json` snippet** are designed in the Plan agent's full output but not included here — Phase B is implemented only if Phase A's smoke test (below) shows the prose is still being ignored.

**Subagent caveat:** `codelens-reviewer` is dispatched as a subagent by the `review` skill. Whether project hooks fire under `claude -p --plugin-dir ... --permission-mode bypassPermissions` for subagents is unverified — this is the open question Phase B implementation must answer first.

## Trade-offs and risks

- **Phase A can still be ignored.** Hardened prose is still prose. Mitigation: the preflight banner's framing ("smoke test greps for these markers") converts the gates into an observable contract — agents honor concretely-falsifiable contracts more reliably than abstract imperatives. Phase B is the actual guarantee.
- **Step 6 depends on `CLAUDE_PROJECT_DIR`.** Verified set inside the ctx_execute sandbox today. If a future context-mode version drops this env var, Step 6 breaks loudly (`Cannot read property 'validateEntry' of undefined`-style error → agent halts with `STATUS: partial`). Acceptable: loud failure is the desired behavior.
- **Step 4 uses `$CLAUDE_PROJECT_DIR` in a shell string.** Same dependency, same loud-failure property.
- **Preflight banner adds ~12 lines to Phase 4.** Agent body grows from 395 to ~420 LOC. Still well under the v0.0.8 spec's `< 320 LOC` target was already missed at 395; this spec does not retarget it.
- **Scorecard column-shape drift (smoke-test Issue #4) is NOT addressed here.** That's a validator-strengthening task (`validate-report.sh` should check the Scorecard has `Severity | Count` columns), separate from "make gates fire."
- **Plugin version drift (0.0.7 in manifests vs 0.0.8 in examples) is NOT addressed here.** Separate version-bump task at release time.
