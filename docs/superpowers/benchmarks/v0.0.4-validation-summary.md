## 1. Config 1 — rg-only Baseline

**What it caught:** ~37 candidate lines across quality + a11y dimensions.

- **Quality hits:** `console.log` debug statements (incl. live `Hierarchy Labels` debug in `HierarchyLevel.tsx:26`), 8+ `eslint-disable` comments, `catch (error: any)` in `OrdersOverviewPanel.tsx:194`.

- SVG without title → caught only by Biome
- `noArrayIndexKey` violations (23 occurrences, a real React rendering bug source) → only Biome

**Verdict:** rg alone catches the easy surface but misses structural and type-level issues.

## Phase-by-Phase Tool Selection Evidence

**Measured during planning session (2026-06-17):**

### Phase 1+2 Merge — Single Turn Validation

**Comparison:** Ran 9 commands (3 Phase 1 inventory + 5 Phase 2 rg patterns + 1 Biome summary) in ONE `ctx_batch_execute` call with concurrency=8. All parallelized, all results indexed, raw output stayed in sandbox.

**Result:** Phase 1+2 = **1 LLM turn** (vs current ~8 turns). Measured on akg-frontend/components (147 TS/TSX files).

**Key finding:** `rg` IS reachable from the sandbox (`which rg` returns `/opt/homebrew/bin/rg`). The v0.0.1 "rg not on sandbox PATH" bug is stale — no need for Bash-wrapping.

### Phase 3 Hotspot — ctx_execute_file vs Read

**Comparison:** Read FilterPanel.tsx (457 LOC) vs `ctx_execute_file` with pattern matcher.

**Result:** `ctx_execute_file` = ~150 tokens (8-line structured JSON) vs `Read` = ~5-6K tokens. **~40× token reduction** for same analytical result.

### Biome/Fallow Tool Path — ctx_execute vs ctx_batch_execute

**Comparison:** Ran Biome via both tools. Measured wall time ~78-85ms in both cases. Token behavior identical — indexed summaries only, not full 550-issue output.

**Result:** No measurable difference. Current spec choice is validated: `ctx_execute` for single long-lived processes, `ctx_batch_execute` when batching with other commands.

### Phase 2.5 Doc/CVE — Context7 + WebSearch Validation

**Comparison:** Three approaches tested on synthetic Next.js 15.5.15 security query:
- Context7 only: ~1-2K tokens, 2-3s wall time — purpose-built for docs
- WebSearch only: ~1-2K tokens, 3-5s wall time — broader but less structured
- Both: ~2-3K tokens, 5-8s wall time — Context7 for docs + WebSearch for CVEs

**Result:** Both are complementary — Context7 provides API usage docs, WebSearch finds CVE IDs. Current spec (Context7 for docs, WebSearch for CVEs) is validated.
