# Report Template (shared)

This is the single source of truth for the Markdown report format. The orchestrator (`agents/codelens-reviewer.md`) applies this template when compiling the final report from `.codelens/findings/*.json`.

## Report Structure

Every codelens report (single-domain or full) follows this structure:

```
# <REPORT_TITLE>

> **Generated by:** codelens v<version> | **Date:** <YYYY-MM-DD> | **Mode:** <single-domain | full | pr>
> **Scope:** <path or commit range> | **Files scanned:** <count>

## Executive Summary

<Narrative — 3-6 sentences. Top severity, top finding type, scope of review, any blockers.>

## Findings by Severity

### Critical
<If none: "No critical findings.">
<For each finding:>

#### <FINDING-ID>: <Title>
- **Severity:** Critical
- **Location:** `<file>:<line-range>`
- **Classification:** <OWASP category or framework tag>
- **Evidence:**
  ```
  <code snippet>
  ```
- **Impact:** <Narrative — what an attacker/user could do>
- **Fix:** <Concrete remediation — code suggestion, link to docs, or step-by-step>

### High
<Same format as Critical>

### Medium
<Same format>

### Low
<Same format>

### Informational
<Same format>

| Positive | Location | Note |
|---|---|---|

## Methodology

| Step | Tool | Notes |
|------|------|-------|
| Scope | <full / path / diff> | <scopeTarget or diffRange if applicable> |
| Domains | <comma-separated requested domains> | |
| Files scanned | <count> | from codelens:scan-meta |
| Pattern extraction | ctx_batch_execute | context-mode (mandatory, auto-indexed by domain) |
| Indexed handoff | ctx_search | reviewers query labeled sources (no disk handoff) |
| Hotspot reads | ctx_execute_file with intent | single-pass (max 15 files, read once by scanner) |
| Library/CVE checks | Context7 | MCP |
| Optional tools | <fallow: detected/skipped> <ast-grep: detected/skipped> | |
| Exclusions applied | <count> | from .claude/codelens-exclusions.json |
```

## Field Rules

- **Severity ordering:** Critical > High > Medium > Low > Informational. Never grouped by domain.
- **Location:** Always `file:line-range`. No bare filenames. No "project-wide".
- **Evidence:** Always a code snippet in a fenced block. Cite exact lines.
- **Impact:** Narrative, not boilerplate. Specific to the finding.
- **Fix:** Actionable. Code suggestion, link to docs, or step list. Not "should be fixed".
- **Cross-domain dedup:** Same file:line (±2) across domains → merge into single finding row, note both classifications.
